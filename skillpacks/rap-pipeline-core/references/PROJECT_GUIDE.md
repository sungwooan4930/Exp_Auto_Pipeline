# RAP 프로젝트 상위 지침

## 목적

이 문서는 Claude Code가 RAP 프로젝트를 수행할 때 따라야 하는 최상위 운영 지침이다.

이 프로젝트는 연구 탐색, 아이디어 구체화, 실험 설계, 시뮬레이션 검증까지를 반복 가능한 문서 기반 workflow로 정리하는 것을 목표로 한다.

## 하이브리드 구조 원칙

이 프로젝트는 전역 코어 스킬팩과 프로젝트 로컬 프로필을 분리한다.

- 코어 스킬팩:
  - `rap-pipeline`
  - 공통 workflow와 단계 로직을 제공
- 프로젝트 로컬 프로필:
  - `RAP_PROJECT_PROFILE.json`
  - 현재 저장소의 주제, 파일 위치, 선호 조건을 제공

현재 저장소의 `claude/` 디렉터리는 코어 스킬팩 초안을 담는 로컬 개발용 소스로 본다.
실제 범용 사용 시에는 설치된 `rap-pipeline` 코어가 `RAP_PROJECT_PROFILE.json`을 읽는다고 가정한다.

## 최상위 운영 구조

전체 workflow는 크게 두 개의 루프로 구성된다.

### A. 연구 서치 루프

흐름:

1. 연구자 요구 입력
2. 논문 탐색
3. 연구자 확인 및 피드백
4. 수정 탐색
5. 연구자 최종 승인까지 반복

핵심 원칙:

- 연구자 승인 전까지 검색 결과를 확정하지 않는다.
- 각 반복은 이전 결과와 피드백을 반영해야 한다.
- 각 반복 결과는 JSON 파일로 저장한다.

### B. 연구 아이디어 구체화 및 검증 루프

흐름:

1. Gap 제시 및 가설 제시
2. 연구자 피드백 반영
3. 연구자 최종 승인까지 반복
4. 실험 디자인 제시
5. 실제 데이터 또는 가상 데이터 기반 시뮬레이션
6. 결과 예측 및 검증
7. 실패 시 Gap/Hypothesis 단계 또는 Experiment Design 단계로 복귀

핵심 원칙:

- Gap과 가설은 함께 다룬다.
- 실험 디자인은 승인된 가설만을 입력으로 사용한다.
- 시뮬레이션 검증 실패 시 원인에 따라 이전 단계로 되돌아간다.

## 단계 구성

### 단계 1. 연구 서치

- 입력: 연구자 요구, 도메인, 키워드, 제약조건, 이전 피드백
- 출력: 정규화된 search JSON
- 반복 조건: 연구자 최종 승인 전까지

### 단계 2. Gap 및 가설 제시

- 입력: 승인된 search 결과, 연구자 피드백, 이전 후보
- 출력: 정규화된 gap 및 hypothesis JSON
- 반복 조건: 연구자 최종 승인 전까지

### 단계 3. 실험 디자인

- 입력: 승인된 gap/hypothesis JSON
- 출력: experiment design JSON

### 단계 4. 시뮬레이션 및 결과 예측

- 입력: experiment design JSON, 실제 데이터 또는 가상 데이터 설정
- 출력: simulation and prediction JSON
- 실패 시: 단계 2 또는 단계 3으로 복귀

## 파일 저장 원칙

- 모든 단계는 JSON 입력과 JSON 출력을 남긴다.
- Markdown은 상위 지침, 설명, 해석, 최종 보고용으로 사용한다.
- 정규화 JSON은 machine-readable한 구조를 유지한다.

## 우선 확인 문서

1. `RAP_PROJECT_PROFILE.json`

4. 해당 단계의 코어 skill 문서

## 완료 정의

- 각 단계는 입력, 처리, 출력, 검증 기준이 문서로 명확해야 한다.
- 연구자 승인 루프가 workflow에 반영되어 있어야 한다.
- 실패 시 되돌아가는 규칙이 문서로 드러나야 한다.
- 각 단계 결과는 정규화 JSON 파일로 재사용 가능해야 한다.
